ROOT CAUSE ANALYSIS
================================================================================

Issue #1: N+1 Query Pattern
  Location: FeedServiceImpl.getCachedFeedItems()
  Problem:  FOR EACH post_id { findById(post_id) }
  Impact:   20 individual Cassandra queries per request
  Solution: Replace with postRepository.findByPostIdIn(List)
  Result:   20 queries → 1 query (20x reduction)

Issue #2: Per-Celebrity Queries
  Location: FeedServiceImpl.getCelebrityFeedItems()
  Problem:  FOR EACH celebrity { findRecentByUserId(celebId) }
  Impact:   50+ individual Cassandra queries for celebrities
  Solution: Replace with celebrityPostRepository.findRecentByUserIds(List)
  Result:   50 queries → 1 query (50x reduction)

Issue #3: Connection Pool Exhaustion
  Location: application.yml cassandra.pool
  Problem:  Default 8 connections, need 20,000+ queries/sec
  Impact:   Queue backlog 250x over capacity = 12.5+ second delays
  Solution: Increase to 32 core, 128 max connections
  Result:   Handles 6x more concurrent users
